## 一、一句话定位（量化版）

> # 「火瞳——帮火锅店每年多赚17万的后厨AI」
>
> 每店每年节流 ¥17.4万（减损¥8万+省人¥9.4万），增收翻台 ¥3.7万。总代一部手机管所有加盟店。

### 定位拆解

| 要素 | 内容 | 量化支撑 |
|------|------|---------|
| **谁** | 火锅店老板/区域总代 | 目标55-60万家火锅店 |
| **做什么** | 后厨AI运营系统 | 损耗检测+SOP合规+翻台分析 |
| **怎么帮** | 视觉AI替代人工巡检 | 4路摄像头+YOLO+VLM+Jetson |
| **省多少** | 年省¥17.4万/店（中位） | 客户验证v2.0 精确测算 |
| **独特性** | 唯一做"运营优化"（非监管） | 竞品v2.0: 所有竞品只做合规 |

## 三、定价策略

### 3.1 定价哲学

> **价值定价，而非成本加成。锚点是客户省下来的钱，不是我们花了多少成本。**

| 定价原则 | 说明 |
|---------|------|
| 锚定节省 | 客户省¥17.4万/年 → 收费¥2.4万 = 客户留¥15万净利 |
| 1/7法则 | 定价 ≤ 总节省的1/7，保证客户ROI > 600% |
| 低于替代品 | 比1个巡检员工资(¥7.2万)便宜67% |
| 高于SaaS | 比美团/客如云贵（¥0.3-2万），但提供他们做不到的价值 |
| 低于安防AI | 比海康AI Box(¥1-3万)便宜或持平，但功能是运营不是安防 |

### 3.2 三阶段定价路径

```
Phase 1: 种子期 (M1-M6，≤10家)
  目标: 快速积累案例+口碑
  定价: ¥12,000硬件 + ¥999/月
  策略: "朋友价"——用低价换案例和数据
  风险: 定价过低→后续提价难

Phase 2: 标准期 (M6-M12，冯校长浙江推广)
  目标: 验证付费意愿+跑通复购
  定价: ¥15,000硬件 + ¥1,999/月
  策略: 标准定价，但给冯校长体系8折
  风险: 价格锚点形成→降价比提价容易

Phase 3: 分层期 (M12+，多客户类型)
  定价:
    单店版 ¥1,499/月 (减配: 仅后厨损耗+桌态)
    标准版 ¥1,999/月 (标配: +SOP+行为)
    专业版 ¥2,999/月 (全功能: +食材+多店总览)
    企业版 ¥4,999+/月 (50+店批量)
  策略: 用单店版降低门槛，用专业版拉高ARPU
```

### 3.3 价格心理锚点

| 锚点 | 客户心理 | 火瞳利用方式 |
|------|---------|------------|
| 1个后厨员工 = ¥72,000/年 | "请人要7万，火瞳只要2.4万" | 在演示中首先对比人力成本 |
| 年食材损耗 = ¥80,000 | "每年浪费8万，花2.4万省回来" | 部署前做一次损耗盘点，让客户"看见"损失 |
| 海康AI Box = ¥20,000+/年 | "海康只做监控都要2万，火瞳还做运营" | 竞品对比表格直接展示 |
| 美团SaaS = ¥6,000-36,000/年 | "美团管前厅收银，火瞳管后厨损耗" | 互补定位，不替代，而是补美团不做的 |

## 四、Go/No-Go 矩阵

### 4.1 决策矩阵

基于5份v2.0文档的综合判断：

| 维度 | 得分(1-5) | 权重 | 加权 | 关键证据 |
|------|:---:|:---:|:---:|------|
| 市场机会 | 5 | 25% | 1.25 | 商业后厨AI渗透<1%，火锅专用AI=0% |
| 客户痛点 | 5 | 20% | 1.00 | 单店年损¥22-33万，总代管控盲区严重 |
| 竞品真空 | 5 | 20% | 1.00 | 所有竞品只做监管不做运营 |
| 技术可行 | 4 | 15% | 0.60 | E2E已验证，火锅专有差距可工程化解 |
| 商业模式 | 4 | 10% | 0.40 | 5店盈亏平衡，ROI强支撑，总代获客高效 |
| 团队能力 | 3 | 10% | 0.30 | 一人可行但需AI辅助，20家后需扩张 |
| **综合得分** | | | **4.55/5** | |
| **Go/No-Go** | | | **✅ GO** | |

> 得分>4.0 = Go, 3.0-4.0 = Conditional Go, <3.0 = No-Go

### 4.2 Go条件与阶段性审查

| 里程碑 | 时间 | Go条件 | No-Go触发 | 决策权重 |
|------|:---:|---------|-----------|:---:|
| **M1 试点运行** | 2026-08 | 冯校长店数据链路稳定>95% | 频繁掉线/误报率高 | 低（修bug） |
| **M3 首付确认** | 2026-10 | ≥1家付费 + 满意度>7/10 | 0付费 + 差评 | 中（验证PMF） |
| **M6 盈亏平衡** | 2027-01 | ≥5家付费 + 月现金流转正 | <3家付费 + 持续亏损 | 🚨 **高** |
| **M12 初具规模** | 2027-07 | ≥20家付费 + 转介绍≥30% | <10家付费 | 高 |
| **M18 区域占领** | 2028-01 | ≥50家 + 浙江品牌案例 | <25家 | 中（评估融资） |

### 4.3 如果No-Go怎么办？

| 阶段 | No-Go场景 | Plan B |
|------|---------|--------|
| M6 No-Go | 飞轮不启动，客户<3家 | 转为冯校长内部工具（自用），停止对外推广，保留技术资产 |
| M12 No-Go | 增长停滞，<10家 | 转型纯SaaS轻模式（方案C），降低获客门槛；或寻求与餐饮SaaS合作 |
| M18 No-Go | 规模不达预期，<25家 | 评估跨品类（中餐/快餐）可能性；或退守冯校长体系做深 |

## 七、竞争应对策略

基于竞品v2.0威胁等级矩阵：

| 竞品场景 | 应对策略 |
|---------|---------|
| **海康推出火锅AI** | 不正面PK硬件价格——强调"我们有火锅数据飞轮(50+店标注数据)，通用模型做不到蒸汽/切配/翻台精度" |
| **政府强制装AI** | 接入政府平台成为合规方案之一，同时保留运营增值模块——"合规免费，省钱付费" |
| **创业公司跟风** | 利用冯校长体系独占+先发12-18个月的时间窗口迅速做大规模；创业公司无行业关系+无数据 |
| **餐饮SaaS集成AI** | 主动寻求合作——"美团/客如云管前厅，火瞳管后厨"，API对接而非竞争 |
| **价格战** | 不降价——用ROI数据证明价值，"你砍价¥500，少省的可能不止¥50,000" |

## 八、定位宣言

> # 「我是做火锅的，不是做AI的。」
>
> **火瞳的价值不在技术本身，在于帮火锅老板看见以前看不见的损失。**
>
> 政府AI盯着后厨有没有违规，火瞳盯着后厨有没有**浪费**。
>
> 不卖监控，卖**利润**。
>
> 一台 Jetson + 四路摄像头 + ¥1,999/月 = 每年多赚 17 万。
>
> **冯校长浙江总代亲测，自己的店先用，效果说话。**
