| **部署方式** | 云边协同：高清监控+天翼云+AI算法 | — |
| **AI能力** | 后厨人员未戴帽/口罩、违规吸烟、鼠患、食材堆放 | 官方方案描述 |

**软肋**:
- ❌ 只做"合规检测"，不做"运营优化"（不管损耗/翻台/效率）
- ❌ 政府项目节奏慢，不灵活
- ❌ 对中小连锁店无专门方案
- ❌ 不提供数据分析Dashboard

### 2.2 海康威视 AI开放平台 + 明厨亮灶

| 维度 | 详情 |
|------|------|
| **成立** | 2001年，安防龙头 |
| **规模** | 年营收900亿+（2024），全球安防第一 |
| **AI能力** | AI开放平台，支持自定义训练。后厨场景有预置算法：厨师帽/口罩/吸烟/鼠患/垃圾桶等 |

| **优势** | 硬件供应链最强、渠道覆盖全国、品牌信任 |
| **软肋** | ❌ 不做"火锅场景"专用模型（蒸汽/蘸料/翻台等） |
| | ❌ 定价偏高，单店 1-3万/年，小连锁难以承受 |
| | ❌ 通用AI，不针对运营指标（损耗率/翻台率/员工SOP） |

## 五、用户口碑扫描

### 5.1 政府AI后厨项目反馈

| 来源 | 反馈 | 含义 |
|------|------|------|
| 深圳"互联网+明厨亮灶" | 合规率<70%→90%+ | **效果显著** |
| 浙江"阳光厨房" | 8万+餐饮单位，合规率91% | 规模化可行 |
| 上海"食安云眼" | 300家试点，投诉下降40% | ROI可量化 |
| 四川校园平台 | 1.2万校覆盖，首月3000次违规 | 快速见效 |
| **商户侧反馈** | 普遍"被装"（政府要求），不主动用 | 无运营价值感知 |
| **消费者侧** | "能看到后厨放心"，但不看Dashboard | 消费端价值有限 |

### 5.2 为什么会存在"火瞳真空"？

```
政府项目（免费）覆盖:
  ✅ 安全合规（帽/口罩/鼠患）
  ❌ 食材损耗（没人管）
  ❌ SOP执行（不属于监管范畴）
  ❌ 翻台效率（不是食安问题）
  ❌ 员工行为（除非涉及食安）
  ❌ 运营数据分析（监管不需要）

→ 火瞳的差异: 做政府"不管"的运营优化
→ 但也意味着: 需要教育市场"AI不止监控"
```
