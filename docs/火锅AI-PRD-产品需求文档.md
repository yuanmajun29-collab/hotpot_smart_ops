## 一、产品定义

### 1.1 一句话

> **店里的第三只眼，总代的决策大脑——让每个加盟店都像总代自己盯着一样干净、规范、赚钱。**

### 1.2 产品形态

Jetson 边缘盒子 + AI 多场景视觉推理 + IoT 传感器 + 云端 Dashboard。

```
门店端（边缘Jetson）           云端（Hub + Dashboard）
 ┌──────────────────┐          ┌──────────────────┐
 │ 摄像头 → YOLO+CLIP│ ─事件──► │ 设备管理+告警分发  │
 │ IoT传感器         │ ◄─配置── │ 多店对比+经营分析  │
 │ 离线自运行        │          │ 区域→门店→设备 层级│
 └──────────────────┘          └──────────────────┘
```

### 1.3 核心差异化

| 护城河 | 说明 |
|--------|------|
| 👁️ 边缘视觉 AI | 前厅+后厨双场景，海底捞刚起步、星巴克完全没有 |
| 📊 层级设备管理 | Zone→Region→Store→Device，连锁扩张零门槛 |
| 💰 已验证 ROI | 冯校长试点 2.5 个月回本、374% 首年回报 |
| 🔌 即插即用 | Jetson 盒子到店插电即用，不需要自建供应链 |

### 1.4 目标用户

| 用户 | 角色 | 核心需求 |
|------|------|---------|
| 区域总代 | 决策者 | 多店总览、数据对比、降本增效 |
| 门店老板 | 使用者 | 后厨管控、前厅效率、食品安全 |
| 店长 | 操作者 | 日常运营、SOP执行、告警处理 |

### 3.4 差别化不作为清单（P3+ · 非当期）

> 火瞳做"盒子"不做"平台"。以下是有意不覆盖的领域：

| 领域 | 理由 | 替代方案 |
|------|------|---------|
| 供应链系统 | 蜀海/哗啦啦已成熟，不是火瞳战场 | 对接主流供应链 API |
| 会员/CRM | 美团/客如云已占领 | 互补合作，非竞争 |
| 政府明厨亮灶合规 | 中国电信垄断·0定价 | 已有摄像头覆盖，不单独做 |
| 收银/点餐 | SaaS红海，无差异化 | 盒子插在收银系统旁 |
