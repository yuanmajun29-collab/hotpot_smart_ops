# 火瞳 — 产品需求文档 (PRD)

> v2.0 | 2026-07-15 | 基于市场调研+冯校长实地验证

---

## 一、产品概述

### 1.1 产品定义

**火瞳**是面向火锅连锁店的AI智能运营系统，通过Jetson边缘盒子+AI视觉+数据看板，覆盖后厨/前厅/食材/员工/管理全场景。

### 1.2 创始背景

冯校长火锅加盟商+浙江区域总代理，打造配套智能化平台，向二级加盟商配送增值服务。

### 1.3 一句话价值

> 「总代一部手机，管好所有加盟店——不在店里，也知道每盘菜去了哪」

### 1.4 目标用户

| 用户 | 角色 | 核心需求 |
|------|------|---------|
| 区域总代 | 决策者 | 多店总览、数据对比、降本增效 |
| 门店老板 | 使用者 | 后厨管控、前厅效率、人力管理 |
| 店长 | 操作者 | 日常运营、SOP执行、告警处理 |

## 二、市场验证

### 2.1 市场规模

| 指标 | 数据 |
|------|------|
| 火锅市场 | 6,175亿元（2024） |
| 全国门店 | 80-100万家 |
| 连锁门店 | ~20万家 |
| 目标客户（3-50店连锁） | ~9万家 |

### 2.2 竞品空白

8款主流餐饮SaaS **全部不做AI视觉后厨**。现有AI玩家聚焦toG明厨亮灶。我们的食材废料检测+浪费追踪+运营数据 = 蓝海。

### 2.3 客户ROI

| 指标 | 数值 |
|------|------|
| 月节省（人工+浪费） | ¥10,500 |
| 首年投入 | ¥26,600 |
| 首年ROI | **374%** |
| 回本周期 | **2.5个月** |

## 三、功能清单

### 3.1 功能总览

```
                   火瞳 智能运营平台
                         │
     ┌────┬────┬────┬────┼────┬────┐
     ▼    ▼    ▼    ▼    ▼    ▼    ▼
   后厨  前厅  食材  员工  SOP  管理  告警
```

### 3.2 P0 功能（MVP·试点必须）

| ID | 模块 | 功能 | 验收标准 |
|----|------|------|---------|
| K01 | 后厨损耗 | 废料实时检测 | YOLO检测→Dashboard推送，延迟<5s |
| K02 | 后厨损耗 | 废料自动计数 | Count Anything计数误差<20% |
| K03 | 后厨损耗 | 每日浪费报告 | 日报自动生成，含趋势图表 |
| K04 | 前厅桌态 | 桌态实时检测 | plan_b <40ms，准确率>90% |
| K05 | 前厅桌态 | 空桌提醒 | 空桌>5分钟自动推送到店长 |
| K06 | 管理总览 | 单店Dashboard | 后厨+前厅数据实时刷新 |
| K07 | 管理总览 | 日报自动推送 | 每日9:00自动发送 |
| K08 | 设备管理 | 硬件注册+心跳 | 盒子在线率>99% |

### 3.3 P1 功能（冯校长全门店·6个月）

| ID | 模块 | 功能 | 验收标准 |
|----|------|------|---------|
| K09 | SOP规范 | 7工位状态机 | 汤底/切配/摆盘/蘸料/洗消/传菜/冷库 |
| K10 | SOP规范 | 操作合规检测 | 违规操作实时告警 |
| K11 | 食材监管 | 进货称重 | 联网电子秤+重量异常告警 |
| K12 | 食材监管 | 以劣充好检测 | CV食材品质对比 |
| K13 | 前厅桌态 | 翻台率分析 | 按时段/桌型统计翻台数据 |
| K14 | 前厅桌态 | 加汤提醒 | 汤位<1/3自动提醒 |
| K15 | 管理总览 | 多店对比Dashboard | 按门店维度横向对比 |
