# 冯校长火锅智能运营 · 决策层一页纸

**Executive Summary · 2026-06-12**

| | |
|---|---|
| **项目** | 冯校长火锅全国连锁智能运营（LLM + VLM + CV + IoT + 边缘计算） |
| **现状** | Phase 0 PoC 已完成，6 条业务闭环可演示 |
| **诉求** | 降本增收 + 食安合规 + 连锁标准化复制 |
| **建议决策** | **批准 Phase 1 立项**：玉环 + 椒江 2 店试点 + 12 周生产化开发 |

---

## 1. 我们要解决什么？

| 痛点 | 根因 | 业务损失 |
|------|------|----------|
| 翻台慢、等位流失 | 桌态信息滞后 | 营收天花板 |
| 食材损耗高、来料短重 | 验收靠人工、无连续数据 | 直接利润损失 |
| SOP 流于形式 | 纸面巡检、无实时校验 | 审计与品牌风险 |
| 冷链/食安隐患 | 温度/门磁未连续监测 | 法律与声誉风险 |
| 加盟难复制 | IT 能力参差、标准不统一 | 扩张速度受限 |

---

## 2. 方案一句话

> **边缘实时感知（CV + IoT）+ 云边智能决策（VLM + LLM）+ 总部中台统管（SOP OTA + 跨店对标）**

五类技术分工：**IoT 给硬数据 · VLM 给品质判断 · CV 给实时状态 · LLM 给运营决策 · 边缘保障低延迟与断网可用**。

---

## 5. 实施路线图（建议批准）

```mermaid
flowchart LR
    P0[Phase0 PoC ✅] --> P1[Phase1 玉环+椒江 3月]
    P1 --> P2[Phase2 区域20店 4月]
    P2 --> P3[Phase3 全国50+店 6月]
```

| 阶段 | 周期 | 范围 | 成功标准 | 预算量级 |
|------|------|------|----------|----------|
| **Phase 1** | 3 月 | 开发 12 周 + 玉环/椒江 2 店试点 | 翻台 +5%；SOP >85% | 研发 6~8 人 + 2 店硬件 |
| **Phase 2** | 4 月 | 区域 20 店 | 来料偏差 <3%；Hub >99.5% | 批量硬件 + 区域 Hub |
| **Phase 3** | 6 月 | 全国 50+ 店 + 加盟 SaaS | 单店部署 <3 天 | 中台 + 运维体系 |

**试点组合**：**玉环店** + **椒江店**（台州区域 2 店并行验证）。

## 6. Phase 1 资源需求

| 类别 | 需求 |
|------|------|
| **人力** | 产品 0.5 + 架构 0.5 + 算法 1~2 + 后端 1~2 + IoT 1 + 前端 0.5~1 + 测试/DevOps/PM 各 0.5 |
| **硬件** | 2 店 × ¥1.2~2.2 万 ≈ **¥2.4~4.4 万** |
| **云/API** | 区域 Hub、对象存储、VLM/LLM 按量 |
| **组织** | 总部 PMO 牵头；区域 IT 驻场；供应链配合 PO 对账 |

---

## 7. 关键风险与对策

| 风险 | 对策 |
|------|------|
| CV 准确率不达标 | 人工复核并行；按店 ROI 标定 |
| POS 接口延期 | 最小字段先行；CSV 日导兜底 |
| 员工抵触 | 首月 KPI 不与告警挂钩；并行过渡期 |
| 加盟 IT 弱 | 预配置镜像 + 4G 必装 + 远程运维 |
