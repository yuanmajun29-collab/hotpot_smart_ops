## 一、邮件版（正式 / PMO 归档）

**主题**：`【评审邀请】冯校长火锅·智能运营 Phase 1 MVP 产品评审（PM-401）`

**收件人**：PMO、研发负责人、试点店长代表（玉环/椒江）、区域督导（可选）、设计（可选）

**正文**：

```
各位好，

兹定于以下时间召开 Phase 1 MVP 产品评审会（PM-401），对试点上线范围、
交互与验收标准进行确认，并形成书面结论。

■ 会议信息
  时间：____年____月____日（周____）____:____ ~ ____:____（约 2 小时）
  形式：□ 现场  □ 腾讯会议  □ 钉钉会议
  地点/链接：________________________________
  主持：________（产品）
  记录：________

■ 会议目标
  1. 确认 Phase 1 产品目标与边界（运营副驾驶，不替代 POS/ERP）
  2. 走查 Must Have 功能（桌态 / IoT / SOP / 成本 / 告警 / 日报 / PDA / 登录）
  3. 确认 UAT 八条阻塞项优先级与 2 周冲刺计划
  4. 输出评审结论：通过 / 有条件通过 / 不通过

■ 议程（120min）
  09:00-09:15  目标与边界（product_goal_card 一页纸）
  09:15-10:15  功能 PRD 走查 + 看板演示（按 F-xxx 逐项）
  10:15-10:45  体验线框：桌态 / PDA / 企微推送 / 手机 H5
  10:45-11:00  七条产品原则合规确认
  11:00-11:15  UAT 阻塞项与 Go-Live 门槛
  11:15-11:30  结论、修订清单、签字

■ 会前准备（请参会人提前 15min 自检）
  【产品】打印/共享 product_goal_card；预约看板环境
  【研发】确保演示环境可用：

  【全员】预读（可选）：docs/product_goal_card.md 第一节「一页纸」

■ 附件 / 链接
  · 产品目标卡：docs/product_goal_card.md
  · 评审勾选表：docs/product_review_checklist.md
  · PRD 摘要：docs/product_design.md §1、§12
  · MVP 验收表：docs/phase1_mvp_acceptance_checklist.md

■ 会后产出（48h 内）
  · 评审结论与修订清单 → product_design_changelog.md
  · 有条件通过项：明确责任人 + 截止日期 + 对应 DEV 号

请回复确认参会。无法参会者请指派授权决策人。

谢谢！
________（产品负责人）
```

## 三、会议当天 · 主持人开场白（3min 口播）

```
各位好，今天是 Phase 1 MVP 的产品评审，不是技术汇报。

我们要回答四件事：
第一，这是不是我们想要的「运营副驾驶」——提醒加确认，不替 POS 收银；
第二，Must Have 功能清单要不要砍、要不要延期；
第三，店长和领班愿不愿意每天用；
第四，哪些项必须现场真数据才能 Go-Live。

评审表在 product_review_checklist，我们按表逐项过。
有演示的马上打开看板，有争议的记到修订清单，最后统一结论和签字。
```
