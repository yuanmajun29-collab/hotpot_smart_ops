## 一、三家全面对比

### 1.1 核心差异

| 维度 | 海底捞 | 星巴克 | 火瞳 (当前) |
|------|--------|--------|:----------:|
| **系统定位** | 服务增强型（技术为人） | 效率驱动型（数据驱动） | 管控增强型（AI替人看店） |
| **覆盖范围** | 中国 1300+ 店 | 全球 38000+ 店 | 试点 1 店 |

| **AI 能力** | 推荐+排班+IoT（弱视觉） | Deep Brew 全场景（零视觉） | 🟢 前厅+后厨视觉AI |

### 1.2 火瞳的护城河

火瞳在三家对比中有 **三项不可替代的优势**：

1. **边缘视觉 AI** — 海底捞刚起步试点，星巴克完全空白。火瞳是唯一覆盖前厅+后厨双场景的。
2. **层级设备管理** — Zone→Region→Store→Device 的设计天然支持连锁扩张，星巴克的 IoT Hub 只到设备级没有区域层。
3. **冯校长验证** — 真实客户、真实痛点、真实 ROI（374%），不凭空造需求。

## 四、产品差异化定位（升级）

### 4.1 vs 海底捞

> 海底捞 AI 是个**百万级的系统**配给自营店。火瞳是**万元级的盒子**配给加盟店。同是火锅，不同战场。

| 海底捞 | 火瞳 |
|--------|------|
| 自建+自用 | 标准化产品 |
| 有蜀海供应链支撑 | 不依赖供应链，盒子即插即用 |
| 重云端、轻边缘 | 边缘视觉AI是核心武器 |
| 1300家直营 | 目标9万家中小连锁 |

### 4.2 vs 星巴克

> 星巴克 Deep Brew 是个**全球统一平台**。火瞳是**每个门店独立的边缘大脑**。星巴克靠数据驱动一致，火瞳靠视觉驱动一致。

| 星巴克 | 火瞳 |
|--------|------|
| 云端AI | 边缘AI |
| 咖啡标准化流程 | 火锅非标服务 |
| App是主通道 | Jetson盒子是主通道 |
| 靠数据预测 | 靠视觉判断 |

### 4.3 核心差异化叙事

```
火瞳的独特价值：

1. 👁️ 看得见 — 边缘视觉AI，两巨头都没有
2. 📊 管得着 — 层级设备管理 + 云端分析，从一家到一千家都能管
3. 💰 回本快 — 2.5个月回本、374%首年ROI，加盟商能算清楚的账
```

```

  context-detector ──→ product ✅

    测试覆盖仅30 → 合理（纯文档产出，非代码交付）
    ↓
Loop 外验证
  red-team-agent ──→ 10项风险(均为项目代码库存量，PRD本身零违规)
  decision-aggregator ──→ 51/100 🟡 准出待审
    → 产品模式+纯文档：低分属预期，无阻断项
    ↓
准出判断
  🟢 准出通过 → PRD v3.0 正式交付
```
