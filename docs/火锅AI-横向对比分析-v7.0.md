## 三、聚类分析：五条技术路线

### 3.1 路线分类

```
路线A: 云端AI驱动 (6家)
  McDonald's · Starbucks · Yum! · Domino's · 瑞幸 · Walmart(部分)
  核心: 推荐引擎 + 需求预测 + 供应链优化
  火瞳关系: 互补。他们做"前端+数据"，我们做"后厨+视觉"

路线B: 厨房机器人 (3家)
  Chipotle · Sweetgreen · Miso
  核心: 机械臂替代人工
  火瞳关系: 不同路线。替代 vs 辅助。昂贵 vs 便宜。

路线C: 边缘视觉 (2家)
  Amazon JWO · ★火瞳
  核心: 摄像头+AI实时分析
  火瞳关系: 同路线，不同战场。零售 vs 火锅后厨。

路线D: 自研重硬件 (1家)
  海底捞
  核心: 配锅机+送餐机器人+IoT
  火瞳关系: 军备竞赛对手。百万 vs 万元。

路线E: 平台型 (1家)
  美团
  核心: 外卖+配送+SaaS
  火瞳关系: 潜在合作伙伴。
```

### 3.2 路线C (边缘视觉) 深度对比

| 维度 | Amazon JWO | 火瞳 |
|------|-----------|------|
| **场景** | 零售收银 | 火锅后厨+前厅 |
| **核心问题** | "谁拿了什么" | "后厨有没有浪费、违规、异常" |
| **摄像头密度** | 200-500个/店 | 4-8个/店 |
| **识别精度** | 99.9%(物品级) | >90%(场景级) |
| **硬件成本/店** | $500K-1M | ¥7K |
| **部署时间** | 数周(装修同步) | <30分钟 |
| **规模化路径** | AWS服务卖给大零售商 | 寄盒子卖加盟连锁 |
| **典型案例** | Sainsbury's(英国超市) | 冯校长火锅 |
| **为何不冲突** | 一个在做"收银革命"，一个在做"后厨革命" |

## 四、SWOT 交叉分析

### 4.1 火瞳 vs 五条路线

| 对比路线 | 火瞳优势(S) | 火瞳劣势(W) | 行业机会(O) | 竞争威胁(T) |
|---------|-----------|-----------|-----------|-----------|
| **A.云端AI** (McDonald's等) | 视觉AI vs 纯数据；不可替代 | 没有推荐引擎/供应链 | 做云端AI巨头的"后厨视觉插件" | 巨头加一个摄像头就能进来 |
| **B.厨房机器人** (Miso等) | ¥7K vs $50K；辅助 vs 替代；风险低 | 不能替代人工操作 | 加盟商买不起机器人但买得起AI | 机器人降价后挤占 |
| **C.边缘视觉** (Amazon) | ¥7K vs $500K；垂直火锅 vs 通用 | 没Amazon的研发资源 | Amazon退出现场→需求验证转向我们 | Amazon重新进入餐饮 |
| **D.自研重硬件** (海底捞) | 产品化 vs 自用；轻 vs 重 | 没海底捞的品牌+资金 | 海底捞验证了"厨房AI有价值" | 海底捞对外卖技术 |
| **E.平台型** (美团) | 深(后厨) vs 广(平台) | 没流量入口 | 与平台互补合作 | 美团自己做厨房AI |

## 七、战略建议

### 7.1 三步走

```
Phase 1 (0-6月): 跑通单店
├── 冯校长30天实测 → 可量化ROI案例
├── 2次模型迭代(蒸汽/油污精度)
└── 标准定价+合同模板

Phase 2 (6-12月): 多店复制
├── 3-5家POC同时跑
├── 加盟商自主下单流程
└── 供应链/美团API对接(可选)

Phase 3 (12-24月): 平台化
├── 100家店→行业基准数据
├── 开放API→第三方集成
└── 关注海康/大华动向
```

### 7.2 四不原则

| 不做 | 理由 |
|------|------|
| 不跟Amazon比视觉精度 | 火锅后厨不需要99.9% |
| 不跟Miso比机器人 | 加盟商用不起 |
| 不跟瑞幸比数据平台 | 云端不是我们的武器 |
| 不跟海底捞比自研 | 我们不造军备，卖工具箱 |

### 7.3 一句话战略

> **在全球餐饮科技版图上，火瞳不跟任何人全面竞争。
> 我们只做一件事：用边缘视觉AI，让加盟连锁的后厨有人盯着。
> 这是所有人都不做的角落，也是9万家中小连锁最痛的角落。**
