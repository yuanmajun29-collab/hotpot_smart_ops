## 一、技术可行性 ✅ 已验证

### 1.1 E2E 数据链路已跑通

```
Mac(YOLO检测) → Jetson(VLM分析:8099) → Hub(:8098) → Dashboard(:3000)
       ✅               ✅                    ✅              ✅
```

验证状态：`E2E_DATAFLOW_TEST: verified` (2026-07-02)

### 1.2 核心技术栈

| 层 | 技术 | 成熟度 | 部署位置 |
|----|------|--------|---------|
| 目标检测 | YOLO26-L | 成熟 | Mac |
| 场景理解 | VLM (Jetson) | 已验证 | Jetson Orin |
| 通用计数 | Count Anything (计划) | 测试中 | Jetson |
| 数据Hub | FastAPI :8098 | 生产就绪 | Mac |
| 面板 | Vue Dashboard :3000 | 生产就绪 | Mac |
| 边缘盒 | Jetson Orin (29GB/12核) | 已连通 | 192.168.2.240 |

### 1.3 技术风险 & 缓解

| 风险 | 等级 | 缓解 |
|------|------|------|
| Jetson GPU资源紧张（跑VLM+计数） | 🟡 | 计划独立Count Anything服务，Docker容器按需启动 |
| 网络延迟（门店→Hub） | 🟡 | 边缘优先推理，Hub端仅收JSON结果 |
| 模型跨场景泛化 | 🟡 | CCL (上下文一致性学习) 可用于训练增强 |
| 蒸汽/遮挡影响检测 | 🟡 | 红外摄像+去雾预处理 |
| Triton依赖（Count Anything） | 🟡 | Docker L4T容器预装triton |
