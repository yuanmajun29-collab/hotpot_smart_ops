# 火瞳 — 产品定位（基于创始背景）

> 版本: v3.0 | 2026-07-15 | 基于冯校长加盟商+浙江区域总代背景

---

## 一、项目背景

**创始身份**: 冯校长火锅加盟商 + 浙江部分区域**总代理**

**核心痛点**: 作为总代，管理多家加盟店的后厨/前厅/员工效率完全靠人，成本不可见、SOP难统一、二级加盟商缺乏管理工具。

**产品初衷**: 打造一套冯校长火锅连锁店的**配套智能化管理平台**，作为向二级加盟商配送的**增值服务**，实现老板随时随地总览全局。

**商业逻辑**:
```
冯校长总代（我们）
    ↓ 配送增值服务
二级加盟商（各门店）
    ↓ 付费使用
火瞳 + 智能硬件
    ↓ 价值回报
降本增效 + 数据决策 + 品牌管控
```

## 四、实施路径

### Phase 1: 试点验证（1家店 · 2个月）

| 里程碑 | 内容 |
|--------|------|
| 选址 | 玉环店 或 椒江店（二选一） |
| 流程梳理 | 采访老板+店长+后厨 → 全流程痛点映射 |
| 硬件部署 | Jetson + 摄像头 覆盖后厨+前厅+进货口 |
| 核心功能上线 | 后厨损耗 + 前厅桌态 + 管理Dashboard |
| 数据采集 | 30天运营数据 → ROI验证 |

### Phase 2: 冯校长全门店（3-10店 · 6个月）

| 里程碑 | 内容 |
|--------|------|
| 首批上线 | 冯校长浙江区域门店全覆盖 |
| 功能扩展 | SOP规范 + 食材监管 + 员工行为 |
| 定价定型 | ¥1,999/店/月 正式定价 |
| 多店管理 | 总代Dashboard — 浙江所有门店一键总览 |

### Phase 3: 标品推广（全餐饮 · 12个月）

| 里程碑 | 内容 |
|--------|------|
| 产品标准化 | 火锅方案 → 通用后厨AI方案 |
| 渠道建设 | 餐饮协会 + 连锁展 + 老客转介绍 |
| 规模目标 | 50+付费门店 |
