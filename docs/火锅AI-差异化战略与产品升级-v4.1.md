## 二、产品升级清单

### 2.1 新增功能 (K31-K32)

| ID | 模块 | 功能 | 吸收来源 | 优先级 | 验收标准 |
|----|------|------|---------|:---:|---------|
| K31 | 后厨品控 | **出品质检** | Domino's DOM | P1 | 每道菜出后厨前YOLO抓拍+CLIP判断(摆盘/份量/异物)，异常<3秒推送到出菜口 |
| K32 | 告警中心 | **动态告警阈值** | Walmart | P1 | 按门店营业时段(午市/晚市/宵夜)自动调整灵敏度，减少高峰误报 |

### 2.2 功能升级 (K24 v2)

| ID | 模块 | 功能 | v3.1 | v3.2 | 吸收来源 |
|----|------|------|------|------|---------|
| K24 | 数据分析 | AI运营建议 | 日报+基础建议 | **可执行建议+效果追踪闭环** | 瑞幸 |

K24 v2 升级：
- 从"你这周浪费率偏高" → "周三晚市牛百叶备货超实际消耗42%，建议减少牛百叶备货至8份/晚，预计周省¥320"
- 新增**效果追踪**：建议执行后，系统自动对比前后7天数据，生成"建议采纳效果报告"

### 2.3 新增定价方案

| 档位 | 硬件 | SaaS | 目标 | 吸收来源 |
|------|------|------|------|---------|
| 试点价 | ¥18,800 | ¥999/月 | 前3家 | 不变 |
| 基础版 | ¥18,800 | ¥7,800/年 | 3-10店 | 不变 |
| 专业版 | ¥25,800 | ¥15,800/年 | 10-50店 | 不变 |
| **🆕 订阅版** | **¥0首付** | **¥1,500/月** | **愿意月付的加盟商** | **Miso模式** |

订阅版不做硬件买卖，盒子作为服务的一部分。降低加盟商决策门槛（¥0首付 vs ¥18,800）。

## 三、架构升级

### 3.1 多摄像头交叉验证管线

```
当前架构                          v3.2 升级
────────                         ────────
Cam_kitchen_01 → YOLO → 告警     Cam_kitchen_01 ┐
                                  Cam_kitchen_02 ├→ Fusion → YOLO → 告警
                                  Cam_kitchen_03 ┘

交叉验证逻辑:
- 单路检测到异常 → "疑似" (降低误报)
- 2+路同时检测到 → "确认" (推送告警)
- 针对蒸汽遮挡场景: 短暂丢失1路不影响判断
```

### 3.2 出品质检管线

```
出菜口摄像头 → YOLO(菜品检测) → CLIP(品质判断) → 出菜口显示屏
                                              → 异常: <3秒推送+红灯
                                              → 正常: 绿灯放行
                                              → 数据: 每小时品控统计到Dashboard
```

### 3.3 动态告警阈值引擎

```
时段自适应:
  午市 11:00-14:00 → 高灵敏度 (人流密集，异常容忍度低)
  下午 14:00-17:00 → 低灵敏度 (备菜期，减少误报)
  晚市 17:00-22:00 → 高灵敏度
  宵夜 22:00-02:00 → 中灵敏度
  深夜 02:00-11:00 → 最低灵敏度 (无人时段)

门店自适应:
  新店(前30天) → 学习期阈值 (宽松，先收集基线)
  稳定店(30天+) → 基于历史数据的个性化阈值
```

## 四、部署标准化 Playbook

### 4.1 30分钟部署SOP

```
00:00  盒子到货 → 店长扫码关注公众号
02:00  关注成功 → 自动推送部署视频(2分钟)
05:00  拆箱 → 插电源 → 盒子开机
08:00  盒子自动扫描局域网IPC → 列出可用摄像头
10:00  微信推送:"检测到4路摄像头，是否全部接入？" → 店长点"是"
12:00  IoT传感器配对 → 微信引导:"请将蓝牙温度计①贴到冷柜1号"
15:00  全部传感器配对完成 → 盒子开始自检
20:00  自检完成 → 微信推送:"火瞳已就绪！正在学习你的后厨..."
25:00  首次数据出现在Dashboard → 微信推送:"第一条检测记录已生成"
30:00  完成 → 30天学习期开始
```

### 4.2 远程支持体系

```
一级: 微信自动引导 (90%问题)
二级: AI客服 (8%问题)
三级: 人工远程 (2%问题·仅需时介入)
```
